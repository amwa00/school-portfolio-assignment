## 📌 Overview
This project is my first portfolio website, built as part of an assignment to practice HTML, CSS, and Bootstrap.

The goal of the project was to create a personal one-page portfolio that includes information about me, my education, experience, projects, and contact details. It was mainly focused on layout, responsiveness, and using Bootstrap components and utility classes.

This is an older version of my portfolio and is no longer used, since I later rebuilt my portfolio from scratch.

## 🚀 Features
- 🏠 One-page layout with multiple sections (Home, About, Experience, Projects, Contact)
- 📱 Responsive design using Bootstrap grid system
- 🎨 Styled with Bootstrap utility classes and custom SCSS
- 👤 Personal “About me” section
- 📚 Education and work experience overview
- 💼 Project showcase section
- 📬 Contact form layout (frontend only)
- 🌐 Smooth section-based navigation

## 🛠️ Tech Stack
- **HTML5**
- **SCSS / CSS**
- **Bootstrap 5**
- **Responsive design principles**

## 🧠 What I learned
In this project I focused on learning the basics of building a structured webpage using a CSS framework.

I worked on:
- Using Bootstrap grid and utility classes for layout
- Building a full one-page portfolio structure
- Making a responsive design that works on different screen sizes
- Organizing content into clear sections
- Combining custom SCSS with Bootstrap styling
- Understanding spacing, alignment, and layout consistency

## 📚 Challenges & Learnings
### Challenges
- Understanding how Bootstrap grid system works
- Keeping layout consistent across all sections
- Balancing custom CSS with Bootstrap utilities
- Making the page responsive without breaking layout structure

### Key Learnings
- How to build layouts faster using Bootstrap
- How responsive design works in practice
- How to structure a personal portfolio page
- How to combine framework styling with custom styling
- Better understanding of spacing and visual hierarchy

## 🌱 Future Improvements (_for this version_)
Since this is an older project, the following improvements would have been made if continued:
- Improve visual design and consistency
- Add JavaScript for interactivity (animations or form handling)
- Improve accessibility (labels, contrast, navigation)
- Refactor SCSS structure for better scalability
- Replace Bootstrap-heavy layout with more custom CSS
